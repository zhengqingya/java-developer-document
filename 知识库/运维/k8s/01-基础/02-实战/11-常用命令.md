# kubectl 常用命令

> 见 https://kubernetes.io/zh-cn/docs/reference/kubectl/generated/kubectl/

```shell
# 打印服务器支持的 API 资源
kubectl api-resources
```

### 命名空间

```shell
# 查看命名空间
kubectl get ns

# 查看默认命名空间`default`下的应用 -- 如果操作不指定命名空间，默认都是放在`default`命名空间下
kubectl get pods
# 查看所有命名空间下的应用
kubectl get pods -A
# 查看指定命名空间`xxx`下的应用
kubectl get pods -n xxx

# 创建命名空间 `zq`
kubectl create ns zq
# 删除 -- 会将其下所有应用删除
kubectl delete ns zq


# 在K8s集群中，对指定命名空间下的 Deployment 执行“无感滚动重启”。 eg：让注入配置生效
kubectl rollout restart deployment demo-java-swck -n zq

# 把 k8s集群里 zq命名空间中 demo-java-swck Service 的 8080端口，映射到你本地 Mac 的 18080端口，让你能在本地直接访问这个服务。
# kubectl port-forward 端口转发。 访问 http://localhost:18080 -> 实际请求会被转发到 demo-java-swck.zq.svc.cluster.local:8080
kubectl port-forward -n zq svc/demo-java-swck 18080:8080 

# 查看最近 20 行日志，并实时跟踪后续新日志
kubectl logs -n zq deploy/demo-java-swck --tail=20 -f

# 查看 zq 命名空间中，标签为 app=demo-java-swck 的 Pod 详细信息
kubectl describe pod -n zq -l app=demo-java-swck
# 查看指定名称的单个 Pod 详情
kubectl describe pod -n zq demo-java-swck-xxx 

```
